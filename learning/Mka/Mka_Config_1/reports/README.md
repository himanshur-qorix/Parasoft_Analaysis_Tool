# Parasoft Reports

Place your Parasoft analysis reports in this directory.

## Required Files

### 1. Parasoft XML Report (Required)
```
ModuleName_parasoft_report.xml
```
- Standard Parasoft output format
- Generated from: `File > Export > Report`
- Contains: All violations, suppressions, metrics

### 2. Suppression Comments (Required)
```
ModuleName_suppress_comments_YYYYMMDD.txt
```
- Extract from source using justification tool
- Format: Same as files in `justifications/` folder
- Contains: All suppression comments with reasons

### 3. Violations JSON (Optional)
```
ModuleName_violations.json
```
- Custom JSON format for easier parsing
- Can be exported from Parasoft or generated by tool
- Contains: Structured violation data

## How to Generate Reports

### From Parasoft GUI

1. Run full analysis on your module
2. Export report: `File > Export > Report`
3. Select format: `XML (Full Report)`
4. Save to this directory

### From Command Line

```bash
# Run Parasoft analysis
cpptest -config "MISRA_2012" -compiler gcc -workspace ./source -report ./reports

# Extract suppressions
python src/interactive_viewer.py --extract-justifications
```

### Using Batch Script

```bash
# Run the complete analysis pipeline
scripts\Build_Parasoft_Rules_Database.bat
```

## File Format Examples

### Suppression Comments Format
```
File: Mka_Main.c
Line: 45
Rules: MISRA2012-RULE-8_9
Reason: Global state required for interrupt-safe operation...
Suppression: parasoft-suppress MISRA2012-RULE-8_9 "Interrupt safety"

---
```

### JSON Violations Format
```json
{
  "module": "Mka",
  "violations": [
    {
      "file": "Mka_Main.c",
      "line": 45,
      "rule": "MISRA2012-RULE-8_9",
      "severity": "Medium",
      "message": "Variable could be declared in smaller scope",
      "suppressed": true,
      "reason": "Global state required for interrupt-safe operation..."
    }
  ]
}
```

## Quality Checklist

Before committing reports:

- [ ] All violations are either justified or fixed
- [ ] Suppression comments are clear and detailed
- [ ] No temporary/debug suppressions included
- [ ] Report matches current source code version
- [ ] File names follow naming convention

## File Naming Convention

```
[ModuleName]_[ReportType]_[YYYYMMDD].[ext]

Examples:
- Mka_parasoft_report.xml
- Mka_suppress_comments_20260422.txt
- Mka_violations.json
- Mka_quality_metrics.json
```

## Size Guidelines

- XML Report: Typically 500KB - 5MB
- Suppression Comments: < 1MB
- JSON Files: < 2MB

Large files indicate:
- Many violations (consider fixing instead of justifying)
- Overly verbose reporting (filter to essential data)

## What AI Learns From

The AI tool analyzes these reports to understand:

1. **Violation Patterns**: Which rules are commonly violated
2. **Justification Quality**: How well violations are explained
3. **Module Context**: Why certain deviations are acceptable
4. **Consistency**: Similar rules justified similarly

## Updating Reports

Update reports when:
- Source code changes significantly
- Coding standards are updated
- New justifications are added
- Parasoft configuration changes

Keep historical versions in `reports/archive/` if needed.
