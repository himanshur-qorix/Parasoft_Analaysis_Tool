# Auto-Generate Workflow

## Overview

The Parasoft AI Analysis Tool now supports **automatic report generation** when `report_dev1.html` is not available. This allows you to analyze source code directly without needing a pre-existing Parasoft report.

## How It Works

### Intelligent Report Detection

When you run the tool, it follows this decision tree:

```
Start Run.bat
     ↓
Prompt for Module Name
     ↓
Prompt for Source Code Path
     ↓
Check: Does report_dev1.html exist?
     ├─ YES → Use Parasoft report
     │         ├─ Source path provided? → Also run MISRA/CERT pre-analysis
     │         └─ No source path → Analyze Parasoft report only
     │
     └─ NO → Check: Source code path provided?
               ├─ YES → Generate MISRA/CERT report automatically
               │         └─ Use generated report for analysis
               │
               └─ NO → ERROR: Need either report or source code

```

### Automatic Report Generation

When `report_dev1.html` is missing but source code path is provided:

1. **Tool runs**: `python src\misra_cert_checker.py <source_path> --report html --output reports\<module>_misra_cert_report.html`
2. **Report is generated**: MISRA-C:2012 and CERT-C violations are detected
3. **Report is used**: The generated HTML report becomes the input for analysis
4. **Full workflow continues**: Qorix mapping, fix generation, knowledge base updates


## Report Format Support

The tool automatically detects and parses:

### Parasoft HTML Reports
- Standard Parasoft violation format
- File-based grouping
- Rule ID mapping

### MISRA/CERT HTML Reports
- Generated by `misra_cert_checker.py`
- Table-based format: Standard | Rule | Severity | Category | File | Line | Message
- Automatic conversion to Parasoft-compatible format

## Configuration

### Enable/Disable Auto-Generation

Edit `config/config.json`:

```json
{
  "analysis": {
    "run_misra_cert_checker": true,
    "misra_cert_report_format": "html"
  }
}
```

- `run_misra_cert_checker`: Set to `false` to disable auto-generation
- `misra_cert_report_format`: Choose `html`, `json`, or `text`

## Error Handling

### Scenario 1: No Report and No Source Code
```
Error: No Parasoft report found and no source code path provided

Please do one of the following:
  1. Place report_dev1.html in the project folder, OR
  2. Provide source code path to generate MISRA/CERT report
```

### Scenario 2: Invalid Source Code Path
```
Warning: Source code path does not exist: D:\Invalid\Path
Skipping MISRA/CERT analysis...

Error: report_dev1.html not found in current directory
Please place the Parasoft report in the tool folder
```

### Scenario 3: Report Generation Failed
```
Error: Failed to generate MISRA/CERT report
# Check source code contains .c/.cpp/.h/.hpp files
# Verify file permissions
```

## Troubleshooting

### Q: Generated report shows 0 violations but source code has issues?

**A:** Check source code path contains analyzable files (.c, .cpp, .h, .hpp)

### Q: Can I customize which violations to detect?

**A:** Edit `src\misra_cert_checker.py` to add/remove rules from `MISRA_RULES` and `CERT_RULES` lists

### Q: Generated report format different from Parasoft?

**A:** The parser automatically converts MISRA/CERT format to Parasoft-compatible structure. All downstream features work identically.

### Q: Can I use JSON or text reports?

**A:** Yes! Change `misra_cert_report_format` in config.json. However, the analysis tool currently requires HTML format for parsing. JSON/text are for manual review.
